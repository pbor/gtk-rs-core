fn activate(&self, parameter: Option<&Variant>)[src]

Activates the action.

parameter must be the correct type of parameter for the action (ie: the parameter type given at construction time). If the parameter type was None then parameter must also be None.

If the parameter GVariant is floating, it is consumed.

parameter

the parameter to the activation

fn change_state(&self, value: &Variant)[src]

Request for the state of self to be changed to value.

The action must be stateful and value must be of the correct type. See [crate::Action::get_state_type()].

This call merely requests a change. The action may refuse to change its state or may change its state to something other than value. See [crate::Action::get_state_hint()].

If the value GVariant is floating, it is consumed.

value

the new state

fn parameter_type(&self) -> Option<VariantType>[src]

Queries the type of the parameter that must be given when activating self.

When activating the action using [crate::Action::activate()], the crate::glib::Variant given to that function must be of the type returned by this function.

In the case that this function returns None, you must not give any crate::glib::Variant, but None instead.

Returns

the parameter type

fn state(&self) -> Option<Variant>[src]

Queries the current state of self.

If the action is not stateful then None will be returned. If the action is stateful then the type of the return value is the type given by [crate::Action::get_state_type()].

The return value (if non-None) should be freed with [crate::glib::Variant::unref()] when it is no longer required.

Returns

the current state of the action

fn state_hint(&self) -> Option<Variant>[src]

Requests a hint about the valid range of values for the state of self.

If None is returned it either means that the action is not stateful or that there is no hint about the valid range of values for the state of the action.

If a crate::glib::Variant array is returned then each item in the array is a possible value for the state. If a crate::glib::Variant pair (ie: two-tuple) is returned then the tuple specifies the inclusive lower and upper bound of valid values for the state.

In any case, the information is merely a hint. It may be possible to have a state value outside of the hinted range and setting a value within the range may fail.

The return value (if non-None) should be freed with [crate::glib::Variant::unref()] when it is no longer required.

Returns

the state range hint

fn state_type(&self) -> Option<VariantType>[src]

Queries the type of the state of self.

If the action is stateful (e.g. created with crate::SimpleAction::new_stateful()) then this function returns the crate::glib::VariantType of the state. This is the type of the initial value given as the state. All calls to [crate::Action::change_state()] must give a crate::glib::Variant of this type and [crate::Action::get_state()] will return a crate::glib::Variant of the same type.

If the action is not stateful (e.g. created with crate::SimpleAction::new()) then this function will return None. In that case, [crate::Action::get_state()] will return None and you must not call [crate::Action::change_state()].

Returns

the state type, if the action is stateful
