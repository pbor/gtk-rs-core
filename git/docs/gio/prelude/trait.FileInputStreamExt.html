fn query_info<P: IsA<Cancellable>>(
    &self,
    attributes: &str,
    cancellable: Option<&P>
) -> Result<FileInfo, Error>[src]

Queries a file input stream the given attributes. This function blocks while querying the stream. For the asynchronous (non-blocking) version of this function, see Self::query_info_async(). While the stream is blocked, the stream will set the pending flag internally, and any other operations on the stream will fail with crate::IOErrorEnum::Pending.

attributes

a file attribute query string.

cancellable

optional crate::Cancellable object, None to ignore.

Returns

a crate::FileInfo, or None on error.

Queries the stream information asynchronously. When the operation is finished callback will be called. You can then call [Self::query_info_finish()] to get the result of the operation.

For the synchronous version of this function, see Self::query_info().

If cancellable is not None, then the operation can be cancelled by triggering the cancellable object from another thread. If the operation was cancelled, the error crate::IOErrorEnum::Cancelled will be set

attributes

a file attribute query string.

cancellable

optional crate::Cancellable object, None to ignore.

callback

user_data
